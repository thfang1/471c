## Minor Extension: Common Subexpression Elimination

### 1) Justification

Programs often contain redundant calculations, either written by the user or generated by previous compiler passes (like array index math).

Benefits users with computation-heavy logic.

### 2) User Viewpoint

```scheme
(let ([a (+ x y)]
      [b (+ x y)])
  (* a b))
```

The compiler detects that `(+ x y)` is computed twice and reuses the result of `a` for `b`.

### 3) Compiler Impact

- **Pipeline:** Added to the Optimization Loop.
- **Mechanism:** Uses an Available Expressions analysis. If an expression has already been computed and its operands haven't changed, replace the second occurrence with the previous result variable.
- **Interaction:** Relies on Uniqification to ensure variable names are stable.

### 4) Success Criteria

- Intermediate L2/L1 code shows a reduction in the total number of arithmetic instructions.
